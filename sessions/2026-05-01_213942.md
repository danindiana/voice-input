### Wizard architecture

Single binary `voice-wizard`. System checks run synchronously before entering raw mode; then ratatui TUI loop at 50 ms poll.

**5 pages:**

| Page | Description |
|------|-------------|
| 1 — Welcome | Title card, mode overview, audio instructions |
| 2 — System Check | GPU/CUDA, whisper model, audio device, voice-input/voice-ambient binaries |
| 3 — Mode Select | ↑/↓ to choose type / print / clip / ambient |
| 4 — Options | Per-mode toggles with live command preview |
| 5 — Launch | Yellow command box; `[C]` copy, `[L]` exec, `[Q]` quit |

**System checks (5 items):**
1. GPU/CUDA — checks `/dev/nvidia0`, falls back to `nvidia-smi`; Warn if absent (CPU fallback OK)
2. Whisper model — `~/.cache/whisper/ggml-large-v3.bin` exists; Fail if missing
3. Audio device — cpal enumeration via `quiet_stderr()` for "pipewire" device; Warn if only default
4. `voice-input` binary — sibling of wizard exe, then PATH `which`; Warn if missing
5. `voice-ambient` binary — same pattern

**Options per mode:**

| Mode | Options |
|------|---------|
| `type` | `--submit` (bool toggle), `--model` (Space to cycle) |
| `print` | `--model` |
| `clip` | `--model` |
| `ambient` | `--no-save` (bool), `--db` (bool), `--model` |

Model cycle order: `large-v3 → large-v2 → medium → small → base`

**`[L] Launch` behaviour:** restores terminal (disable raw mode, leave alternate screen) then `exec()` replaces the wizard process with `sh -c "<assembled command>"` — the voice-input program inherits a clean TTY.

**Navigation:** `→`/Enter = next page, `←` = back, `↑`/`↓` = select, `Space` = toggle option, `q`/`Esc` = quit anywhere, `Ctrl+C` = quit anywhere.
